Overview

The Document Processing API allows users to upload documents (PDF, DOCX, scanned images) and receive extracted text and structured data.

Phase 0 Endpoints

  • GET /status: Health check. Returns the operational status of the API.
  • POST /process/document/: Accepts a file upload for processing.
    • File: `multipart/form-data`
    • Query Parameter: `output_format` (e.g., 'text', 'json')
    • Note: Full processing logic is not yet implemented in Phase 0.
  • GET /: Serves the API testing page.

Future Phases Will Include:

  • Specific processing logic for DOCX, PDF (text-based), and scanned documents (OCR).
  • Text cleaning and normalization.
  • Table extraction.
  • Image extraction and tagging.
  • Various output formats (plain text, JSON, Markdown, CSV for tables).
